iContainer

A native macOS app for Apple's container CLI

Manage containers, images, logs, stats, and an interactive shell through a clean SwiftUI interface — built on top of the official container CLI, with on-device AI log analysis that never leaves your Mac.

Download for macOS View on GitHub

Features

Everything the CLI does, with a native interface

Pulled straight from the app's real feature set — no vaporware.

🧠

AI log analysis, on-device

One Explain click on any container's or machine's Logs tab turns raw output into a concise diagnostic: a health summary, the errors and warnings it found (quoted), the likely cause, and concrete next steps — streamed live. Powered by Apple's Foundation Models, so the model runs entirely on your Mac: your logs — secrets, tokens, connection strings and all — never leave the device. No cloud, no API keys, no subscription. Requires Apple Intelligence on a supported Mac.

📦

Container lifecycle

Create from an image or a Dockerfile build, start, stop, restart, edit settings (ports, volumes, environment), and delete.

🖥️

Container machines

Manage the Linux VMs that host containers: create, start/stop, edit CPUs, memory, and home mount, delete — with Info, Shell, and Logs tabs per machine.

🗂️

Images & registry

List, pull, inspect, and delete images, with registry login and guided error handling for authentication failures.

📊

Info, Stats, Shell & Logs

Per-container detail tabs: network/mounts/DNS/env in Info, live charts in Stats, a persistent Shell session, and Logs with follow mode.

🔄

Automatic updates

iContainer keeps itself current via Sparkle: it checks in the background and installs new versions in place, with your confirmation and signature verification.

🎛️

Menu bar extra

Control containers, machines, and the Apple container service without opening the main window.

⌨️

Keyboard shortcuts

A full menu bar command set — ⌘N new container, ⇧⌘P pull image, ⌘1–⌘4 detail tabs, and more.

Also included

  • Dashboard — counts at a glance and quick actions
  • Sidebar — reorderable sections, search, filters, live status
  • Apple container service — start/stop, status, version, stats, live logs
  • Notifications — optional alerts on stop or failure
  • Settings — theme, launch at login, polling cadence, default shell, terminal font, custom CLI path, default registry

Install

Get iContainer

Homebrew keeps you on the latest release automatically — or grab the DMG directly.

Install via Homebrew

The easiest way to stay up to date.

# add the tap, then install the cask
brew tap nico81/icontainer
brew install --cask icontainer

# later, to upgrade:
brew upgrade --cask icontainer

Or download the signed & notarized .dmg from the latest GitHub release and drag iContainer into Applications.

Requirements

What you need before you start.

  • ✓ macOS 26 or later (Apple silicon)
  • ✓ Apple Container CLI installed
  • ✓ Apple Intelligence enabled Optional — only needed for on-device AI log analysis; everything else works without it.
